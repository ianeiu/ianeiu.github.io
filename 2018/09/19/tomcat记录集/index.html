Tomcat——记录集

作者： 吴炜棉 / 2018-09-19 / 分类： Tomcat

配置, 问题

修改startup.bat启动窗口名

打开 catlina.bat 文件，修改title

:doStart  
shift  
if "%TITLE%" == "" set TITLE=zsyx_8232
set _EXECJAVA=start "%TITLE%" %_RUNJAVA%  
if not ""%1"" == ""-security"" goto execCmd  
shift  
echo Using Security Manager  
set "SECURITY_POLICY_FILE=%CATALINA_BASE%\conf\catalina.policy"  
goto execCmd

tomcat指定jdk

  • Windows环境
    打开 catlina.bat 文件、增加配置
    set JAVA_HOME=C:\Program Files\Java\jdk1.7.0_79

  • linux环境
    编辑 setclasspath.sh，在文件的最开始加入如下代码
    set JAVA_HOME=/home/jdk/Java\jdk7\jdk1.7.0_51
    set JRE_HOME=/home/jdk/Java\jdk7\jre7

占用8080端口问题

tomcat死机了，重启eclipse之后，发现

Several ports (8080, 8009) required by Tomcat v6.0 Server at localhost are already in use.  
The server may already be running in another process, or a system process may be using the port.   
To start this server you will need to stop the other process or change the port number(s).

解决办法：
方法1.重启电脑
方法2.在dos下，输入

netstat   -ano|findstr  8080   
//说明：查看占用8080端口的进程  
taskkill  /pid  6856  /f   
//说明，运行windows自带taskkill命令，将上面显示的进程号，结束掉。
